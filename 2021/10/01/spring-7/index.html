Spring - Validation

Posted on 2021-10-01 Edited on 2021-10-05 In java , spring Views:

Spring

Validation

  • Validation이란 에외를 방지하기 위한 검증하는 과정을 뜻한다
  • if (id == null) || (passwd == null) { return } 과 같은 코드를 통해 처리 할 수있지만
    비지니스 로직에서 벗어난 코드들이 작성되게 되고 이러한 검증코드들 때문에 전체적인 로직이
    작동하지 않을 수 있다
  • 따라서 분리해서 작성해야하는데 이를 스프링에서 anntation을 통해 관리할 수 있다
  • Validator 에서 확인 할 수 있다

validation 활용하기

  • gradle dependency 에
    implementation 'org.springframework.boot:spring-boot-starter-validation' 을 추가한다

  • 다음과 같은 요청이 들어왔지만 email의 형태도 phone nubmer의 형태도 이상하지만 검증이 없이 데이터가 echo되었다.

  • Validation 을 하기 위해 dto의 필드에 상황에 맞는 annotation을 작성한다

  • 그리고 검증이 필요한 객체에대해 @Valid anntation을 붙여주도록한다
  • 같은 리퀘스트를 보내면

  • Bad Request를 응답한다

  • 정상적인 메일형식을 요청하면 정상적인 응답을 볼 수 있다

  • 정규식 을 이용한 Validation을 할 수도 있다

Validation Handling

  • validation에 통과하지 못한 요청에대하여 따로 핸들링 할 수 있다

  • 메소드에 BindingResult 라는 객체를 파라미터를 받음으로써 Validation의 결과를 받을 수 있다.

  • 정상적으로 요청했을 경우엔 Echo이지만 정규식을 통과하지 못한경우 다음과 같이 응답하도록 할 핸들링 할 수 있다.

  • 여러 Annotation을 통해 validation할 수 있다

Custom Validation

  • Spring에서 제공하는 기본적인 Validation Annotation외에 필요한 경우 직접 규칙을 만들어서 사용할 수 있다

  • @AssertTrue 또는 @AssertFalse annotation을 통해 메소드를 작성한다

  • 요청으로 yyyyMM 형식으로 날짜 데이터를 받는다고 가정하자 이때 @Size 어노테이션 적용할 경우
    글자수만 맞으면 통과 되므로 다른 방법을 사용해야한다

  • 같은 요청에 대하여 Bad Req를 응답한것을 볼 수 있다

  • 요구사항에 맞게 req했을 때 정상적으로 작동하는것을 확인할 수 있다

  • 하지만 이렇게 작성할 경우 재사용이 불가능하다 이를 해결하기위해 Annotation을 custom 할 수 있다

Custom Annotation

  • 기존에 존재하는 annotation도 요구사항에 맞춰 custom이 가능하다
  • 해당 annotation을 사용할 validator class 를 구현
  • ConstraintValidator 인터페이스를 구현한다

Spring Boot - Get

Posted on 2021-09-13 Edited on 2021-09-16 In spring , java

Spring Boot

GET API

  • get은 리소스를 취득하기 위한 메소드 CRUD에서 R에 해당된다

  • @GetMapping을 통해 주소 매핑시 cmd + b 단축키를 통해 어노테이션에 들어갈 수 있는 것들이 정의되어 있다.

  • 다른 방식으로도 맵핑이 가능하다

  • 위 방식처럼 매핑했을때 hi 라고 요청할 경우 get,post,put, delete와 같은 모든 메소드에 request 되기 때문에 method를 명시해줘야 한다.

  • 따라서 다음과 같이 수정하여 사용 할 수 있다.


pathvariable

  • 변화하는 path에 대하여 일대일로 메소드를 작성하는것은 매우 비효율 적이다.

  • 사용자에따라 변화 할 수 있는 값(id, 이름 등)에 대하여 변수로 받기위한 것이다.

  • 반드시 GetMapping과 PathVariable의 변수명은 같아야한다

  • 그러나 개발중 path variable과 method의 파라미터 명을 다르게 해야할 때는 다음과 같이 사용 할 수있다.

Query Parameter

  • 구글에서 Spring Boot를 검색했을 때 URL주소는 다음과 같다

  • URL을 봤을때 ?뒤에 Key Value쌍으로 이루어진 것을 볼 수있다

  • ?Key = value & Key2 = value2 & Key3 = value 3…

  • Mapping을 위해서 @RequestParam Anntation을 추가해야 정상적인 매핑이 가능하다

  • 위의 방식대로하게되면 맵에 접근하기위해서는 get(key)명령어를 통해서 접근해야한다

  • 명시적으로 파라미터 이름을 정해줄 수 있다

  • 3개의 파라미터 (name , email, age) 뿐만아니라 파라미터에 갯수가 늘어날때마다 그것에대하여 모두 대응되는 코드의 작성이 필요하다

  • @RequestParam String grade , @RequestParam String address …

  • 이러한 불편함을 해소하기 위해 DTO (Data Trnsfer Object)를 정의하여 컨트롤 할 수 있다

  • 코드가 훨씬 간단해졌다
  • 이떄는 @RequestParam을 붙이지 않는다
  • QueryParameter 주소에 들어있는 ?name=정이수&email=mustdoit2020@gmail.com&age=28 키를 해당 Spring 내부에서 UserRequest객체의 변수에 key와 매핑시켜준다

Strategy Pattern

  • 전략 패턴은 유사한 행위들을 캡슐화 하여 객체의 행위를 바꾸고 싶은 경우 직접변경하는 것이 아닌

  • 개방폐쇄 원칙과 의존 역전 법칙을 따른다

  • Encoder를 사용하는 클래스에서 어떤 전략인지 설정만하면 코드 수정 없이 사용할 수 있다

Facade Pattern

  • 구조 패턴
  • 정면, 앞면이라는 뜻을 가지는 Facade는 여러 개의 객체와 실제 사용하는 서브 객체 사이에 복잡한 의존관계가 있을때 중간에 Facade 객체를 두어 제공하는 interface만을 활용하여 기능을 제공하는 방식이다
  • Facade는 자신이 가지고 있는 각 클래스의 기능을 명확히 알아야 한다

OOP - Design Pattern - Proxy

Posted on 2021-09-06 In java , oop , design pattern

Proxy Pattern

  • 구조 패턴
  • 일반적으로 프록시는 다른 무언가와 이어지는 인터페이스의 역할을 하는 클래스이다. 프록시는 어떠한 것(이를테면 네트워크 연결, 메모리 안의 커다란 객체, 파일, 또 복제할 수 없거나 수요가 많은 리소스)과도 인터페이스의 역할을 수행할 수 있다.
  • Client는 Proxy로 부터 결과를 받는다

  • 현재 proxy가 없는 코드에서는 볼때마다 계속해서 로딩을 하게되므로 큰 오버헤드가 발생 할 수있다.

  • 로딩은 한번만 진행되고 그 이후엔 캐싱을 통해 오버헤드를 줄일 수 있다

aop

  • 원하는 기능들, 흩어져있는 기능들을 동일하게 제공하는 역할을 프록시 패턴을 이용하여 만들 수 있다
  • 특정한 메소드 실행시간, 메소드 전후로 특정한 작업을 하거나 어떠한 리퀘스트나 리스폰 정보를 남긴다던지 일괄적으로 패키지에 모든 메소드들에 기능을 제공 할 수있다
